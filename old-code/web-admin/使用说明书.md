## 1. 系统概述

本系统是扑克游戏平台的 **Web 管理后台**，供平台运营人员对用户、游戏、财务、内容等进行统一管理。

| 项目 | 说明 |
|------|------|
| 前端框架 | Vue 3 + CoreUI |
| 状态管理 | Pinia |
| 构建工具 | Vite |
| 登录接口 | `http://43.254.167.52:11001/pp-login` |
| 管理接口 | `http://43.254.167.52:11001/pp-admin` |
| 实时通信 | WebSocket `ws://43.254.167.52:11001/pp-chat-1` |

## 2. 环境要求与启动

### 2.1 本地开发启动

```bash
# 安装依赖
npm install

# 启动开发服务器（默认 http://localhost:5173）
npm run dev

# 构建生产包
npm run build
```

### 2.2 API 地址切换

API 地址统一在 `src/Config.ts` 中配置：

```ts
export const Config = {
    loginUrl: 'http://43.254.167.52:11001/pp-login',
    adminUrl:  'http://43.254.167.52:11001/pp-admin',
    wsUrl:     'ws://43.254.167.52:11001/pp-chat-1',
    timeout:   30000,
    retryTimes: 3
}
```

> 切换测试/生产环境时，直接修改上方三个 URL 即可，注释部分为备用 HTTPS 地址。

## 3. 登录系统

1. 打开浏览器，访问后台地址。
2. 在登录页输入**管理员账号**和**密码**，点击「登录」。
3. 登录成功后，系统会将 Token 保存至浏览器本地存储（键名：`admin_token`），并自动跳转到主界面。
4. 若长时间未操作导致 Token 失效（错误码 `99`），系统会自动跳转回登录页，重新登录即可。

> **注意：** 权限不足时系统返回错误码 `{$n}`，请联系超级管理员授权。

### 5.3 道具配置

**路径：** 配置 → 道具配置（`/admin/config/item`）

配置游戏内道具的名称、价格、图标、使用效果等属性。

**常用操作：**
- **新增道具**：点击「新增」，填写道具信息后保存。
- **修改道具**：在列表中点击对应行的「编辑」按钮。
- **禁用道具**：切换状态开关为关闭，用户将无法购买或使用该道具。

---

### 5.4 抽水配置

**路径：** 配置 → 抽水配置（`/admin/config/water`）

配置平台从各类游戏场次中抽取的佣金比例（抽水率）。

> **重要：** 修改抽水比例会直接影响平台收益及玩家实际所得，修改前请确认无误。

---

### 5.5 充值奖励配置

**路径：** 配置 → 充值奖励配置（`/admin/config/charge-reward`）

设置充值档位对应的奖励方案，例如充值 100 元赠送 10 金币等。

**操作步骤：**
1. 查看当前奖励档位列表。
2. 点击「新增」添加新档位，或点击「编辑」修改现有档位。
3. 填写充值金额、赠送比例/固定金额，启用后生效。

## 6. 管理模块

### 6.1 用户管理

**路径：** 管理 → 用户管理（`/admin/users`）

展示平台所有注册用户的信息列表。

**主要功能：**

| 功能 | 说明 |
|------|------|
| 搜索用户 | 支持按用户 ID、用户名、手机号等条件检索 |
| 查看详情 | 点击用户行查看账号基本信息、余额、注册时间等 |
| 修改用户 | 可调整用户昵称、状态（启用/禁用/封号） |
| 调整余额 | 手动增减用户账户余额，需填写操作原因 |
| 强制下线 | 踢出当前在线用户的游戏会话 |

---

### 6.2 注册审核

**路径：** 管理 → 注册审核（`/admin/users/register-audit`）

当登录配置中开启了「邀请码注册」或「审核制注册」后，新用户注册申请会进入此列表等待审核。

**操作流程：**
1. 列表中显示待审核的用户注册申请。
2. 点击「通过」：用户账号激活，可正常登录。
3. 点击「拒绝」：输入拒绝原因，通知申请人。

---

### 6.3 匹配管理

**路径：** 管理 → 匹配管理（`/admin/groups`）

查看和管理当前正在进行匹配的玩家组，可强制解散匹配队列或调整匹配参数。

---

### 6.4 房间管理

**路径：** 管理 → 房间管理（`/admin/rooms`）

查看所有游戏房间的实时状态，包括房间 ID、当前人数、游戏进度等。

**主要功能：**
- 查看房间列表及在线玩家。
- 强制关闭异常房间。
- 查看房间历史牌局记录。

## 7. 比赛模块

### 7.1 比赛管理

**路径：** 比赛 → 比赛管理（`/admin/matches`）

创建和管理平台举办的各类扑克比赛（锦标赛/坐庄赛等）。

**创建比赛步骤：**
1. 点击「新建比赛」。
2. 填写比赛名称、开始时间、买入金额、奖励池设置等。
3. 保存后比赛进入「未开始」状态，到达开始时间后自动开放报名。
4. 比赛结束后在此页面查看结果并发放奖励。

---

### 7.2 聊天-客服 客服不跟比赛有关，暂时放在这里的

**路径：** 比赛 → 聊天-客服（`/admin/chat`）

实时查看平台内的全局聊天记录，支持：
- 搜索关键词过滤聊天内容。
- 删除违规消息。
- 禁言特定用户。

---

### 7.3 邮件管理

**路径：** 比赛 → 邮件管理（`/admin/mail`）

向平台用户发送系统邮件（站内信），常用于活动通知、奖励发放等。

**发送步骤：**
1. 点击「新建邮件」。
2. 选择发送对象：全体用户 / 指定用户 ID。
3. 填写邮件标题、正文内容，可附带金币/道具奖励。
4. 点击「发送」，邮件将推送至用户收件箱。

## 8. 新闻模块

### 8.1 新闻管理

**路径：** 新闻 → 新闻管理（`/admin/news`）

管理平台新闻公告，内容将展示在 App 的新闻/公告页面。

**操作：**
- **发布新闻**：填写标题、正文（支持富文本），设置发布时间后保存。
- **下架新闻**：将状态切换为「隐藏」，App 端不再展示。
- **置顶**：启用置顶后，该条新闻展示在列表最顶部。

---

### 8.2 横幅管理

**路径：** 新闻 → 横幅管理（`/admin/news/banners`）

管理 App 首页轮播横幅（Banner）。

**操作：**
1. 上传横幅图片（建议尺寸按 App 端规范）。
2. 设置跳转链接（可选）。
3. 设置展示顺序（数字越小越靠前）和有效期。
4. 切换启用/停用状态控制是否展示。

## 9. 机器人模块

**路径：** 机器人 → 机器人管理（`/admin/robot`）

管理平台内的 AI 机器人账号，用于在游戏中填充桌位、维持游戏流量。

本模块包含多个子功能面板：

| 子面板 | 说明 |
|--------|------|
| **机器人面板** | 查看所有机器人的在线状态、当前游戏状态 |
| **全局控制** | 一键上线/下线所有机器人，设置机器人总数上限 |
| **个性配置** | 配置机器人的行为风格（激进/保守/随机），胜率参数等 |
| **比赛计划** | 设置机器人参与特定比赛的报名计划 |
| **匹配计划** | 配置机器人加入普通匹配房间的时机和条件 |
| **充值商店** | 模拟机器人的账户资金情况 |
| **流水查询** | 查询机器人账号的资金流水记录 |

> **注意：** 机器人数量和行为参数直接影响游戏体验，请根据实际在线人数合理配置，避免机器人比例过高。

## 11. 充值 / 提现模块

### 11.1 链信息管理

**路径：** 充值/提现 → 链信息（`/admin/charge/chains`）

管理平台支持的区块链网络信息，例如 TRC20、ERC20 等。

**操作：**
- 新增支持的链（填写链名称、链 ID、合约地址等）。
- 修改或禁用已有链配置。

---

### 11.2 充值配置（区块链）

**路径：** 充值/提现 → 充值配置-区块链（`/admin/charge/config-blockchain`）

配置各区块链通道的充值规则：
- 最低充值金额。
- 平台收款地址（钱包地址）。
- 手续费率。
- 是否启用该通道。

---

### 11.3 提现配置（区块链）

**路径：** 充值/提现 → 提现配置-区块链（`/admin/charge/withdraw-config-blockchain`）

配置用户提现到区块链地址的规则：
- 最低/最高提现金额。
- 每日提现次数上限。
- 手续费（固定金额或比例）。

---

### 11.4 充值 / 提现审核

**路径：** 充值/提现 → 充值/提现审核（`/admin/charge/orders`）

处理用户的充值确认和提现申请。

**充值审核流程：**
1. 用户不需要提交充值审核，直接向Cregis提供的充值地址转账
2. Cregis收到转账交易后会回调给后台
3. 后台收到回调自动上帐

**提现审核流程：**
1. 用户申请提现后，订单显示在列表中（状态：待审核）。
2. 点击「审核」查看用户信息和提现金额。
3. 确认无风险后点击「通过」，系统将自动或人工向用户钱包转账。
4. 如有异常（欺诈风险、余额不足等），点击「拒绝」。

> **安全提示：** 审核大额提现前，请核查用户账号是否有异常充值/游戏行为，防范洗钱风险。

---

### 11.5 奖池管理

**路径：** 充值/提现 → 奖池管理（`/admin/charge/pot`）

查看和管理游戏奖池资金，包括：
- 当前奖池总额查看。
- 手动向奖池注入或抽取资金（特殊活动使用）。
- 查看奖池历史变动记录。
