```cpp
bool registerFilePickerIOS(se::Object* obj) {
    obj->defineFunction("openFilePickerIOS", _SE(js_openFilePickerIOS));
    return true;
}

static bool js_openFilePickerIOS(se::State& s) {
    const auto& args = s.args();
    if (args.size() == 1 && args[0].isString()) {
        std::string mimeType = args[0].toString();
        openFilePickerIOS(mimeType.c_str());
        return true;
    }
    return false;
}
SE_BIND_FUNC(js_openFilePickerIOS)
```

## 🐛 已知问题与限制

### Android
- ⚠️ content://类型URI需要复制文件到缓存，大文件可能较慢
- ⚠️ Android 6.0+需要运行时权限请求
- ⚠️ 某些Android版本的文件选择器UI不同

### iOS
- ⚠️ 需要完成C++到JavaScript的绑定
- ⚠️ 需要测试不同iOS版本的兼容性
- ⚠️ 需要处理iCloud文件

### Web
- ⚠️ 某些旧浏览器不支持File API
- ⚠️ 大文件可能导致内存问题
