# 文件选择功能使用说明

## 概述

已经为Cocos Creator项目添加了跨平台的文件选择功能，支持：
- ✅ Web浏览器平台（原有功能）
- ✅ Android Native平台（已实现）
- 🔧 iOS Native平台（已预留接口，需要完善）

## 功能特性

### 1. 统一的API接口
无论在哪个平台，都使用相同的API调用方式：

```typescript
import { FileUtils } from "./core/utils/FileUtils";

// 选择任意文件
const result = await FileUtils.selectLocalFile(this.node, "*/*");

// 选择图片文件
const result = await FileUtils.selectLocalFile(this.node, "image/*");

// 选择特定类型文件
const result = await FileUtils.selectLocalFile(this.node, "image/png");
```

### 2. 返回结果格式

```typescript
type SelectLocalFileRet = {
    uri: string;           // 文件URI（Web）或路径（Native）
    ext: string;           // 文件扩展名，如 "jpg", "png"
    blob: Blob;            // 文件内容（Blob对象）
    filePath?: string;     // 原生平台：文件绝对路径
    fileName?: string;     // 原生平台：文件名
}
```

### 3. 支持的方法

#### `selectLocalFile(node, accept?)`
选择文件并返回完整信息（包括Blob对象）

```typescript
const file = await FileUtils.selectLocalFile(this.node, "image/*");
if (file) {
    console.log("文件路径:", file.filePath);
    console.log("文件名:", file.fileName);
    console.log("扩展名:", file.ext);
    console.log("Blob大小:", file.blob.size);
}
```

#### `selectLocalFileWithFile(node, accept?)`
选择文件并返回File对象（主要用于Web平台）

```typescript
const file = await FileUtils.selectLocalFileWithFile(this.node, "*/*");
if (file) {
    console.log("文件名:", file.name);
    console.log("文件大小:", file.size);
}
```

#### `selectLocalFileBase64(node, accept?)`
选择文件并转换为Base64编码

```typescript
const result = await FileUtils.selectLocalFileBase64(this.node, "image/*");
if (result) {
    console.log("扩展名:", result.ext);
    console.log("Base64:", result.base64);
    // 可以直接用于上传到服务器
}
```

## 平台实现详情

### Android平台

#### Java层实现
文件：`AppActivity.java`

**主要方法：**
```java
// 打开文件选择器
public static void openFilePicker(String mimeType)

// 处理选择结果
protected void onActivityResult(int requestCode, int resultCode, Intent data)
```

**调用方式：**
```typescript
// TypeScript/JavaScript会自动调用
jsb.reflection.callStaticMethod(
    "org/cocos2dx/javascript/AppActivity",
    "openFilePicker",
    "(Ljava/lang/String;)V",
    mimeType
);
```

#### C++层JNI绑定
文件：`FilePicker_JNI.cpp`

**功能：**
- 接收Java层的回调
- 将结果传递给JavaScript层
- 函数签名：`Java_org_cocos2dx_javascript_AppActivity_nativeOnFilePickerResult`

**编译配置：**
需要在`CMakeLists.txt`或`Android.mk`中添加：
```cmake
# CMakeLists.txt
list(APPEND GAME_SOURCE
    ${CMAKE_CURRENT_LIST_DIR}/../../../jni/FilePicker_JNI.cpp
)
```

或者在`Android.mk`中：
```makefile
LOCAL_SRC_FILES := FilePicker_JNI.cpp
```

### iOS平台（预留）

#### Objective-C++实现
文件：`FilePicker_iOS.mm`

**主要组件：**
- `FilePickerDelegate`: 处理文档选择器回调
- `openFilePickerIOS()`: C++函数，打开文件选择器

**集成步骤：**
1. 将`FilePicker_iOS.mm`添加到Xcode项目
2. 在C++层注册`openFilePickerIOS`函数到JavaScript
3. 在`Info.plist`中添加必要的权限声明

**需要的权限（Info.plist）：**
```xml
<key>UISupportsDocumentBrowser</key>
<true/>
```

### Web平台
保持原有的HTML5 File API实现，无需额外配置。

## 使用示例

### 示例1：选择图片并显示

```typescript
import { FileUtils } from "./core/utils/FileUtils";

@ccclass
export default class ImagePicker extends cc.Component {
    
    @property(cc.Sprite)
    targetSprite: cc.Sprite = null;
    
    async onPickImageClick() {
        try {
            // 选择图片文件
            const result = await FileUtils.selectLocalFile(this.node, "image/*");
            
            if (!result) {
                console.log("用户取消选择");
                return;
            }
            
            console.log("选择的文件:", result.fileName);
            
            // 显示图片（Web平台）
            if (result.uri && !cc.sys.isNative) {
                const spriteFrame = await FileUtils.loadTextureByUri(result.uri);
                if (spriteFrame) {
                    this.targetSprite.spriteFrame = spriteFrame;
                }
            }
            
            // 显示图片（Native平台）
            if (result.filePath && cc.sys.isNative) {
                cc.loader.load(result.filePath, (err, texture) => {
                    if (!err) {
                        const spriteFrame = new cc.SpriteFrame(texture);
                        this.targetSprite.spriteFrame = spriteFrame;
                    }
                });
            }
            
        } catch (error) {
            console.error("选择文件失败:", error);
        }
    }
}
```

### 示例2：上传文件到服务器

```typescript
import { FileUtils } from "./core/utils/FileUtils";

@ccclass
export default class FileUploader extends cc.Component {
    
    async onUploadClick() {
        try {
            // 选择文件并转换为Base64
            const result = await FileUtils.selectLocalFileBase64(this.node, "*/*");
            
            if (!result) {
                console.log("用户取消选择");
                return;
            }
            
            // 上传到服务器
            const response = await fetch("https://your-server.com/upload", {
                method: "POST",
                headers: {
                    "Content-Type": "application/json",
                },
                body: JSON.stringify({
                    fileName: result.fileName,
                    fileExt: result.ext,
                    fileData: result.base64,
                })
            });
            
            if (response.ok) {
                console.log("上传成功!");
            } else {
                console.error("上传失败:", response.statusText);
            }
            
        } catch (error) {
            console.error("上传文件失败:", error);
        }
    }
}
```

### 示例3：读取文本文件内容

```typescript
import { FileUtils } from "./core/utils/FileUtils";

@ccclass
export default class TextFileReader extends cc.Component {
    
    @property(cc.Label)
    contentLabel: cc.Label = null;
    
    async onReadTextClick() {
        try {
            // 选择文本文件
            const result = await FileUtils.selectLocalFile(this.node, "text/*");
            
            if (!result) {
                console.log("用户取消选择");
                return;
            }
            
            // 读取文本内容
            const reader = new FileReader();
            reader.onload = (e) => {
                const text = e.target.result as string;
                this.contentLabel.string = text;
                console.log("文件内容:", text);
            };
            reader.readAsText(result.blob);
            
        } catch (error) {
            console.error("读取文件失败:", error);
        }
    }
}
```

| JPEG图片 | `"image/jpeg"` |
| 所有文本 | `"text/*"` |

| 所有视频 | `"video/*"` |
| 所有音频 | `"audio/*"` |

## 注意事项

### Android平台
1. **权限配置**：确保在`AndroidManifest.xml`中有读取外部存储的权限（Android 6.0+需要运行时权限）
2. **文件访问**：选择的文件会被复制到应用缓存目录，确保有足够的存储空间
3. **JNI编译**：确保`FilePicker_JNI.cpp`被正确编译链接

### iOS平台
1. **待完善**：iOS实现需要进一步测试和完善
2. **权限声明**：需要在`Info.plist`中添加文档访问权限
3. **C++绑定**：需要将`openFilePickerIOS`函数绑定到JavaScript环境

### Web平台
1. **浏览器兼容性**：部分旧浏览器可能不支持File API
2. **安全限制**：只能选择用户主动触发的文件
3. **内存管理**：大文件可能导致内存问题，注意释放资源

## 故障排查

### Android平台问题

**问题1：调用后没有反应**
- 检查JNI函数是否正确编译
- 查看logcat日志是否有错误信息
- 确认`AppActivity`类路径正确

**问题2：无法读取文件内容**
- 检查文件权限
- 确认文件路径正确
- 查看是否是content://类型的URI（会自动复制到缓存）

**问题3：回调未触发**
- 确认`onNativeFilePickerResult`全局函数已定义
- 检查C++的JNI回调是否正确执行
- 查看控制台是否有错误信息

### iOS平台问题

**问题1：文件选择器无法打开**
- 确认`FilePicker_iOS.mm`已添加到项目
- 检查`openFilePickerIOS`函数是否正确绑定
- 查看Xcode控制台日志

**问题2：选择后崩溃**
- 检查delegate是否正确设置
- 确认内存管理正确（ARC）
- 查看崩溃日志
