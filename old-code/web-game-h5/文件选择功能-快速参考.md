# 文件选择功能 - 快速参考

## 📁 修改的文件

### 1. Android Java层
**文件**: `build/jsb-link/frameworks/runtime-src/proj.android-studio/app/src/org/cocos2dx/javascript/AppActivity.java`

**新增内容**:
- `openFilePicker(String mimeType)` - 打开文件选择器（静态方法，供JS调用）
- `onActivityResult()` - 处理文件选择结果
- `getFilePathFromUri()` - 将URI转换为本地路径
- `getFileNameFromUri()` - 获取文件名
- `nativeOnFilePickerResult()` - JNI回调声明

### 2. TypeScript工具类
**文件**: `assets/scripts/core/utils/FileUtils.ts`

**新增内容**:
- 平台检测函数：`isNativeAndroid()`, `isNativeIOS()`, `isWeb()`
- `selectLocalFileNativeAndroid()` - Android原生实现
- `selectLocalFileNativeIOS()` - iOS预留实现
- `selectLocalFileWeb()` - Web平台实现（原有逻辑）
- `readNativeFile()` - 读取原生文件辅助函数
- `arrayBufferToBase64()` - ArrayBuffer转Base64
- 全局回调：`window.onNativeFilePickerResult`

**修改内容**:
- `selectLocalFile()` - 改造为跨平台自动路由
- `selectLocalFileWithFile()` - 添加原生平台支持
- `selectLocalFileBase64()` - 优化原生平台处理
- `SelectLocalFileRet`类型 - 添加`filePath`和`fileName`字段

### 3. Android C++ JNI绑定（新建）
**文件**: `build/jsb-link/frameworks/runtime-src/proj.android-studio/app/jni/FilePicker_JNI.cpp`

**功能**:
- 实现JNI回调函数：`Java_org_cocos2dx_javascript_AppActivity_nativeOnFilePickerResult`
- 将Java层结果转发到JavaScript层
- 调用全局回调`window.onNativeFilePickerResult`

### 4. iOS Objective-C++实现（新建，预留）
**文件**: `build/jsb-link/frameworks/runtime-src/proj.ios_mac/ios/FilePicker_iOS.mm`

**功能**:
- `FilePickerDelegate` - 文档选择器代理
- `openFilePickerIOS()` - C++函数，打开文件选择器
- 处理iOS文档选择结果
- 调用全局回调`window.onNativeFilePickerResult`

## 🚀 快速使用

### 基础调用
```typescript
import { FileUtils } from "./core/utils/FileUtils";

// 选择任意文件
const result = await FileUtils.selectLocalFile(this.node, "*/*");

// 选择图片
const result = await FileUtils.selectLocalFile(this.node, "image/*");

// 转换为Base64
const base64Result = await FileUtils.selectLocalFileBase64(this.node, "image/*");
```

### 处理结果
```typescript
if (result) {
    console.log("文件名:", result.fileName);      // Android/iOS
    console.log("文件路径:", result.filePath);    // Android/iOS
    console.log("扩展名:", result.ext);           // 所有平台
    console.log("Blob:", result.blob);            // 所有平台
}
```

## 🔧 需要完成的集成步骤

### Android平台 ✅

**步骤1**: 编译JNI代码
在`build/jsb-link/frameworks/runtime-src/proj.android-studio/app/jni/CMakeLists.txt`中添加：
```cmake
list(APPEND GAME_SOURCE
    ${CMAKE_CURRENT_LIST_DIR}/FilePicker_JNI.cpp
)
```

**步骤2**: 权限配置（如需要）
在`AndroidManifest.xml`中添加：
```xml
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />
```

**步骤3**: 重新构建项目
```bash
cd build/jsb-link/frameworks/runtime-src/proj.android-studio
./gradlew assembleDebug
```

### iOS平台 🔧

**步骤1**: 添加文件到Xcode项目
- 打开`.xcodeproj`项目
- 将`FilePicker_iOS.mm`添加到项目中

**步骤2**: 绑定C++函数到JavaScript
在合适的初始化代码中添加：
```cpp
se::ScriptEngine::getInstance()->evalString(
    "window.openFilePickerIOS = function(mimeType) { "
    "  /* 调用C++的openFilePickerIOS函数 */ "
    "};"
);
```

**步骤3**: Info.plist配置
```xml
<key>UISupportsDocumentBrowser</key>
<true/>
```

## 🐛 常见问题

### Q1: 调用后没有反应
**A**: 检查以下几点：
1. JNI代码是否编译
2. 查看logcat日志
3. 确认类路径正确：`org/cocos2dx/javascript/AppActivity`

### Q2: 回调未触发
**A**: 确认：
1. `window.onNativeFilePickerResult`已定义（自动定义）
2. C++层正确调用JavaScript函数
3. 查看控制台错误信息

### Q3: 无法读取文件内容
**A**: 检查：
1. 文件权限（Android 6.0+需要运行时权限）
2. 文件路径正确性
3. content:// URI会自动复制到缓存
